## 2. 技术栈选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 原生 JS（vanilla JS） | 无框架依赖，所有页面为静态 HTML |
| HTTP 请求 | XMLHttpRequest | 原生 AJAX，无框架 |
| 后端 | Node.js + Express | Web 框架 |
| 数据库 | SQLite（sql.js） | 纯 JS 实现的 SQLite，零配置 |
| 密码哈希 | bcryptjs | 纯 JS 实现，无编译依赖 |
| 文件上传 | multer | Express 文件上传中间件 |
| Session | express-session | 服务端 session 管理 |
| 模板渲染 | 服务端无模板引擎 | 前端自己拼 HTML（字符串拼接） |

## 3. 模块职责

### 3.1 后端模块

| 模块 | 文件 | 职责 |
|------|------|------|
| 应用入口 | `server/app.js` | 配置中间件、注册路由、启动服务 |
| 数据库层 | `server/db/database.js` | 所有 SQL 操作封装 |
| 清单路由 | `server/routes/lists.js` | 清单 CRUD + 搜索 API |
| 留言路由 | `server/routes/messages.js` | 留言提交 + 审核 + 回复 API |
| 分类路由 | `server/routes/categories.js` | 分类查询 + CRUD API |
| 认证路由 | `server/routes/auth.js` | 管理员登录/登出 API |
| 发布人路由 | `server/routes/publishers.js` | 发布人管理 + 登录验证 API |
| 设置路由 | `server/routes/settings.js` | 站点设置 API |
| 认证中间件 | `server/middleware/auth.js` | 管理员 session 校验 |

### 3.2 前端模块

| 模块 | 文件 | 职责 |
|------|------|------|
| API 封装 | `api.js` | 封装所有后端 API 调用 |
| 共用工具 | `common.js` | 页头/页脚渲染、清单模板、留言模板、工具函数 |
| 全局样式 | `command.css` | 所有页面样式（前台 + 后台） |

## 4. 请求流程示例

```
访客浏览清单列表：
  浏览器 → GET /index.html → Express 返回静态 HTML
         → 浏览器解析 HTML → 加载 api.js / common.js
         → DOMContentLoaded → Common.init()
         → API.getLists(1,30) → GET /api/lists?page=1&size=30
         → Express 路由 → database.getPublishedLists()
         → SQLite 查询 → 返回 JSON
         → JavaScript 渲染 HTML → 用户看到列表
```

```
发布人提交清单：
  浏览器 → 填写表单 → submitList() → API.createList()
         → POST /api/lists → Express 路由
         → authenticatePublisher() 验证账号密码
         → createList() 写入数据库（status='pending'）
         → 返回 201 → 前端显示成功
```

## 5. 安全设计

| 威胁 | 防御措施 |
|------|----------|
| SQL 注入 | 全部使用参数化查询（? 占位符） |
| XSS | escapeHtml() 转义 & < > " |
| 未授权访问 | requireAdmin 中间件校验 session |
| 文件上传恶意文件 | 白名单 .jpg/.jpeg/.png/.gif + 随机文件名 |
| 密码泄露 | bcrypt 10轮哈希存储 |
| Session 劫持 | Cookie httpOnly + sameSite=lax |
