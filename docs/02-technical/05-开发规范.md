# 开发规范

> 版本：1.0.0 | 最后更新：2026-05-24

---

## 1. 编码规范

### 1.1 语言

- 前端：原生 JavaScript（ES5），不引入框架
- 后端：Node.js（CommonJS 模块，`require/module.exports`）
- 样式：纯 CSS，不引入预处理器

### 1.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | 全小写 + 连字符 | `database.js`、`lt-post.html` |
| 变量 | 驼峰 | `listTitle`、`productCode` |
| 函数 | 驼峰 | `renderListTable()`、`submitList()` |
| 全局对象 | 首字母大写 | `Common`、`API`、`UPLOAD_CONFIG` |
| 数据库表 | 蛇形复数 | `list_items`、`admin_users` |
| 数据库列 | 蛇形 | `product_code`、`display_name` |
| CSS 类/ID | 驼峰或连字符 | `#infoText`、`.message-reply` |
| API 路径 | 全小写 + 斜杠 | `/api/admin/lists/search` |

### 1.3 代码格式

- 缩进：2 个空格
- 引号：单引号（JS）/ 双引号（HTML）
- 分号：JS 每行尾加分号
- 大括号：K&R 风格（左大括号在行末）
- 行宽：不超过 120 字符

```javascript
// 正确示例
function getListById(id) {
  runSql("SELECT * FROM lists WHERE id=?", [id]);
  saveToFile();
}
```

### 1.4 注释规范

- 每个文件顶部写文件注释（功能说明）
- 每个函数/方法写行注释（用途 + 参数说明）
- 复杂逻辑写块注释
- 不写显而易见的注释（如 `// 定义变量`）

```javascript
// 文件顶部
// ============================================================
// 清单 API 路由
// 职责：处理所有与清单相关的 HTTP 请求
// ============================================================

// 函数注释
// 分页获取已发布的清单列表（前台使用）
// @param {number} page - 页码，从1开始
// @param {number} size - 每页条数
// @returns {object} { rows, total, totalPages }
```

## 2. API 规范

- 统一前缀 `/api`
- 统一响应格式 `{ ok: boolean, data?, error? }`
- 成功状态码：200（GET）、201（POST）
- 错误状态码：400（参数错误）、401（未登录）、403（无权限）、404（不存在）、409（冲突）
- URL 参数使用 snake_case：`page_size`、`product_code`

## 3. 数据库规范

- 表名、字段名使用 snake_case
- 主键统一为 `id INTEGER PRIMARY KEY AUTOINCREMENT`
- 所有时间字段类型为 TEXT，存储格式 `YYYY-MM-DD HH:MM:SS`
- 使用 CHECK 约束限制枚举值
- 所有 SQL 使用参数化查询（? 占位符），禁止拼接 SQL

## 4. 安全规范

- 密码必须使用 bcrypt 哈希存储（10 轮）
- 所有用户输入在输出到 HTML 前必须转义 `escapeHtml()`
- 上传文件必须：白名单校验扩展名 + 限制文件大小 + 随机文件名
- Session Cookie 设置 `httpOnly` 和 `sameSite`
- API 接口中的管理员操作必须通过 `requireAdmin` 中间件

## 5. Git 规范

```
分支命名：
  main          — 主分支（可部署版本）
  dev           — 开发分支
  feature/xxx   — 功能分支
  fix/xxx       — 修复分支

提交信息格式：
  feat: 新增功能说明
  fix:  修复问题说明
  refactor: 重构说明
  docs: 文档更新
  chore: 构建/工具更新
```
