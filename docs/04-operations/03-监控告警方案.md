# 监控告警方案

> 版本：1.0.0 | 最后更新：2026-05-24

---

## 1. PM2 监控

PM2 提供了基础的进程监控功能。

```bash
# 查看进程状态
pm2 status

# 实时监控面板（CPU/内存）
pm2 monit

# 查看日志
pm2 logs mylistblog

# 查看最近200行日志
pm2 logs mylistblog --lines 200
```

## 2. 系统资源监控

### 2.1 磁盘空间

```bash
# 查看磁盘使用
df -h

# 查看项目目录大小
du -sh /var/www/mylistblog/data/
du -sh /var/www/mylistblog/uploads/
```

### 2.2 内存使用

```bash
# 查看 Node.js 内存使用
pm2 monit

# 查看系统内存
free -h
```

### 2.3 CPU 和负载

```bash
# 查看系统负载
top

# 查看 Node.js CPU 使用
pm2 monit
```

## 3. 应用健康检查

### 3.1 手动检查

```bash
# 检查 API 是否正常
curl -s http://localhost:3000/api/lists | head -c 100

# 检查页面是否正常
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/index.html
```

### 3.2 自动监控脚本

创建一个简单的健康检查脚本 `healthcheck.sh`：

```bash
#!/bin/bash
# 健康检查脚本 — 如果 API 连续3次返回非200，发送告警

URL="http://localhost:3000/api/lists"
COUNT=0

for i in 1 2 3; do
  STATUS=$(curl -s -o /dev/null -w "%{http_code}" $URL)
  if [ "$STATUS" != "200" ]; then
    COUNT=$((COUNT+1))
  fi
  sleep 5
done

if [ "$COUNT" -ge 3 ]; then
  echo "服务异常！状态码连续3次非200" | mail -s "mylistblog 服务告警" your-email@example.com
fi
```

配合 crontab 每5分钟执行一次：

```bash
*/5 * * * * /var/www/mylistblog/healthcheck.sh
```

## 4. 日志管理

PM2 日志默认位置：`~/.pm2/logs/`

```bash
# 查看日志文件路径
pm2 show mylistblog | grep -E "out log path|error log path"

# 设置日志自动轮转（按大小）
pm2 install pm2-logrotate
pm2 set pm2-logrotate:max_size 10M
pm2 set pm2-logrotate:retain 7
```

## 5. 告警阈值建议

| 指标 | 警告 | 严重 |
|------|------|------|
| 磁盘使用率 | > 80% | > 90% |
| 内存使用率 | > 80% | > 90% |
| CPU 使用率 | > 70% 持续10分钟 | > 90% 持续5分钟 |
| API 响应时间 | > 1秒 | > 3秒 |
| 进程状态 | — | 停止运行 |
