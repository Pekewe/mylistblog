# 数据备份与恢复策略

> 版本：1.0.0 | 最后更新：2026-05-24

---

## 1. 需要备份的数据

| 数据 | 位置 | 备份方式 |
|------|------|----------|
| 数据库 | `data/mylistblog.db` | 文件复制 |
| 上传图片 | `uploads/images/` | 目录复制 |

**不需要备份：**
- `node_modules/` — 可通过 `npm install` 重新安装
- 代码文件 — 可通过 Git 或源码包恢复

## 2. 备份方案

### 方案A：手动备份（推荐）

```bash
# 停止服务
pm2 stop mylistblog

# 备份数据库和图片
cp /var/www/mylistblog/data/mylistblog.db /backup/$(date +%Y%m%d)-mylistblog.db
cp -r /var/www/mylistblog/uploads/images /backup/$(date +%Y%m%d)-images/

# 恢复服务
pm2 start mylistblog
```

### 方案B：定时自动备份（crontab）

```bash
# 编辑 crontab
crontab -e

# 加入以下内容（每天凌晨3点备份）
0 3 * * * cp /var/www/mylistblog/data/mylistblog.db /backup/$(date +\%Y\%m\%d)-mylistblog.db && cp -r /var/www/mylistblog/uploads/images /backup/$(date +\%Y\%m\%d)-images/

# 保留最近30天的备份（可选）
0 3 * * * find /backup/ -name "*.db" -mtime +30 -delete
```

## 3. 恢复流程

```bash
# 1. 停止服务
pm2 stop mylistblog

# 2. 恢复数据库文件
cp /backup/20260524-mylistblog.db /var/www/mylistblog/data/mylistblog.db

# 3. 恢复图片
cp -r /backup/20260524-images/* /var/www/mylistblog/uploads/images/

# 4. 重启服务
pm2 start mylistblog
```

## 4. 备份策略建议

| 环境 | 频率 | 保留周期 | 方式 |
|------|------|----------|------|
| 开发环境 | 按需 | — | 手动 |
| 生产环境 | 每天 | 30天 | crontab 自动 |

## 5. 注意事项

- 备份前先停止服务，确保数据库文件一致
- 上传图片和数据库需同时备份，否则图片无法显示
- sql.js 的数据库文件在运行时可能处于不一致状态，停止服务后再备份最安全
