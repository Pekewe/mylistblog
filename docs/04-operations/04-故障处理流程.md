# 故障处理流程

> 版本：1.0.0 | 最后更新：2026-05-24

---

## 故障处理通用流程

```
发现问题
    │
    ▼
1. 确认影响范围（前台/后台/全部）
    │
    ▼
2. 查看日志 → pm2 logs mylistblog
    │
    ▼
3. 根据下方具体故障处理
    │
    ▼
4. 验证修复
    │
    ▼
5. 记录到故障日志
```

## 常见故障

### 故障1：服务无法启动

**现象：** `node server/app.js` 报错退出

**排查步骤：**

```bash
# 1. 检查 Node.js 版本
node --version   # 需要 >= 18

# 2. 检查依赖是否安装
ls node_modules/express

# 3. 如果依赖缺失
cd server && npm install

# 4. 检查端口是否被占用
lsof -i :3000
# 如果被占用，杀掉进程或修改 PORT

# 5. 检查数据库目录权限
ls -la data/
```

### 故障2：页面打开但数据加载失败

**现象：** 页面正常显示，但列表显示「加载中…」或「加载失败」

**排查步骤：**

```bash
# 1. 检查 API 是否正常
curl http://localhost:3000/api/lists
# 如果返回连接失败 → 服务未启动或端口不对

# 2. 检查特定 API
curl http://localhost:3000/api/lists/1
# 如果返回 {} 或错误 → 检查 database.js

# 3. 打开浏览器开发者工具（F12）→ Network 标签
# 查看哪个 API 请求失败 → 复制 URL 到新标签页测试
```

### 故障3：管理员登录失败

**现象：** 输入正确账号密码仍提示错误

**排查步骤：**

```bash
# 1. 确认数据库中有管理员记录
curl http://localhost:3000/api/auth/me
# 如果返回 {"ok":true,"data":null} → 未登录

# 2. 检查 session 配置
# 查看 app.js 中的 session 配置是否正确

# 3. 如果密码忘记
# 停止服务 → 删除 data/mylistblog.db → 重启
# 密码重置为 admin123
```

### 故障4：发布清单失败

**现象：** 提交清单时提示错误

**排查步骤：**

```
1. 确认发布人账号密码正确
   → 在后台「发布人管理」检查账号状态是否为「启用」

2. 确认必填字段已填写
   → 清单名称、分类、内容不能为空

3. 检查后端日志
   → pm2 logs mylistblog
```

### 故障5：图片上传失败

**现象：** 选择图片后上传无反应或报错

**排查步骤：**

```bash
# 1. 确认管理员已登录
# 2. 检查文件大小是否超过 5MB
# 3. 检查文件格式是否符合（jpg/png/gif）
# 4. 检查 uploads 目录是否存在且可写
ls -la uploads/images/

# 5. 如果目录不存在
mkdir -p uploads/images
```

### 故障6：留言提交后不显示

**现象：** 提交成功，但留言板看不到

**原因：** 留言默认为「待审核」状态，需要管理员审核通过后才会显示。

**解决：** 登录后台 → 留言管理 → 点击「通过」

### 故障7：数据库损坏

**现象：** 启动时提示数据库错误

**原因：** SQLite 文件损坏（极少发生，通常在异常关机时）

**解决：**

```bash
# 1. 尝试从备份恢复
cp /backup/最新备份.db /var/www/mylistblog/data/mylistblog.db

# 2. 没有备份时，删除数据库重新初始化
# 注意：这将丢失所有数据！
rm /var/www/mylistblog/data/mylistblog.db
pm2 restart mylistblog
# 重新启动后种子数据会自动填入
```
